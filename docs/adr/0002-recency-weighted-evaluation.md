# ADR 0002: Recency-Weighted Evaluation

- Status: Accepted
- Date: 2026-04-04

## Context

ABC は年々重くなっており、古い `E` を解けることと最近の `E` を自力で解けることは同義ではない。
過去問を広く解いているほど、古い成功体験で現在地を過大評価しやすい。

## Decision

- 実力評価の主指標は「最近 25 件の記録済み ABC」とする
- ユーザーがその時点の最近学習主戦場を明示した場合のみ、補助データとして低めの重みで加える
- 特定ディレクトリを恒久的な補助指標として固定しない
- 異種問題群を含む学習ディレクトリは、数値的な補助指標と質的な育成指標に分けて扱う
- 数値 blend は比較可能な subset にだけ使う
- 直近 75 件は 25 件ずつに区切って傾向を見る
- コメント付きコードは「解説依存の近似 proxy」として使う

## Consequences

- 古い `E/F` の成功だけで今の frontier を判断しなくなる
- 今その時点の学習実態を、固定ルールにせず snapshot として取り込める
- JOI のような総合練習を、ABC 文字難易度に無理に潰さず重く評価できる
- 「解けた本数」より「最近の自力率」を重視できる
- コメント有無は完璧ではないが、継続観測には十分有用
